step8: 課金導入・会員状態管理・Android公開準備

目的:
現場ノート Flutterアプリに課金機能を導入し、
無料 / 100円 / 500円 のプラン制御を実装したうえで、
Androidで内部テスト〜公開申請できる状態まで整備してください。

前提:
- Flutter + Riverpod + SQLite ベース
- アプリ名は「現場ノート」
- ローカルファースト構成
- 有料ユーザーのみクラウド連携を前提
- 課金は Flutter の in_app_purchase パッケージを利用
- Android先行リリース
- iOS対応は今回は不要
- 500円プランには初月無料トライアルを想定
- スマホ機能は全プランで大きく変えず、差分は保存方法・閲覧制限・WEB/クラウド機能で表現する

プラン仕様:
1. 無料プラン
- 保存: ローカル
- 履歴表示: 制限あり
- 検索: 簡易
- 地図/一括編集/WEB連携: 制限ありまたは不可

2. 100円プラン（LOCAL）
- スマホ内で履歴無制限
- 写真無制限（ローカル）
- 保存: ローカルのみ
- バックアップなし
- 機種変更引き継ぎなし

3. 500円プラン（CLOUD）
- スマホ機能は100円プランと同等
- クラウド同期あり
- バックアップあり
- WEB利用可能
- フル検索
- 地図全履歴
- 一括編集
- 月次集計
- 初月無料トライアル対応前提

実装要件:
- in_app_purchase の導入
- 課金商品IDを定数管理
- 購入状態を永続化する Subscription/Entitlement レイヤーを実装
- 起動時に課金状態を復元する
- restore purchase を実装
- プラン状態を Riverpod provider で全画面から参照可能にする
- free / local / cloud の enum を作る
- 課金状態に応じた feature gate を実装する
- UI上で未購入機能に触れた時は自然に課金導線を表示する
- 51件目到達、検索実行時、機種変更導線でアップグレード訴求を表示できる構成にする
- 500円プラン向けの「クラウド同期準備済み」のインターフェースを用意する
  - まだ本実装が未完でも、同期サービスの抽象化までは行う
- 将来の Supabase Auth / Sync 追加に備えた設計にする
- デバッグ用に現在の課金状態を確認できる hidden debug panel を追加する

必要な画面/導線:
- 設定画面に「プラン」セクション追加
- 現在のプラン表示
- プラン比較画面
- 100円プラン購入ボタン
- 500円プラン購入ボタン
- 購入復元ボタン
- 購入中 / 購入完了 / エラー表示
- 無料ユーザー向けアップグレード訴求ダイアログ
- 500円プラン向け WEB/クラウド説明セクション

設計要件:
- 課金SDKへの依存をUIから分離する
- billing_service, entitlement_service, paywall_controller など責務分離する
- mock 実装に差し替えやすくする
- 商品IDは environment / constants に分離
- 例外時もアプリが落ちないようにする
- オフライン時は最後に確定した権限状態を参照する
- 将来的な Android / iOS 両対応を見据えた抽象化にする

Android公開準備:
- Android Manifest / package 名 / app name / icon / versioning を確認
- release build が通るように整備
- Proguard/R8 で致命的な問題が出ない構成にする
- 署名設定用の雛形ファイルや手順コメントを追加
- README に以下を追記
  - Google Play Console で必要な作業
  - 課金商品登録手順
  - internal testing の流れ
  - 署名設定手順
  - リリースビルド方法
- アプリ内に最低限の以下文言の導線を入れる
  - 利用規約（プレースホルダ可）
  - プライバシーポリシー（プレースホルダ可）
  - お問い合わせ先（プレースホルダ可）

完了条件:
- アプリが起動する
- release build が通る
- 無料 / 100円 / 500円 の状態分岐が実装済み
- 課金状態の復元処理がある
- プラン比較画面がある
- 課金導線が複数箇所に配置されている
- Android公開に必要なREADMEと設定ガイドが追加されている

出力してほしいもの:
- 追加/変更ファイル一覧
- 実装内容の要約
- 未接続部分（Play Consoleの商品IDや署名キーなど）
- 手元で確認すべき手順
